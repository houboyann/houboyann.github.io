Chapter1 CMOS组合逻辑门设计

S1.1 静态CMOS设计

  • 每一时刻每个门的输出通过一个低阻路径连到 VDDV_{DD}VDD​ 或者 VSSV_{SS}VSS​ 上
  • Output是电路实现的布尔函数

S1.1.1 静态互补CMOS

  1. 静态CMOS门是上拉网络（PUN）和下拉网络（PDN）的组合

  3. PUN由PMOS组成，PDN由NMOS组成：
    • NMOS强“0”弱“1”，PMOS强“1”弱“0”（Threshold Drop）

  4. 数字电路的优点：
    • 高噪声耐性（模拟电路中SNR信噪比，数字电路中NM噪声容限）

S1.1.3 传输门逻辑（PTL）

  1. 传统互补CMOS的另一种选择
  2. 优点：更少的晶体管；没有静态功耗
    缺点：NMOS弱“1”，在 VDDV_{DD}VDD​ 时性能很差
  • Exp3.1 AND Gate
  1. 传输门电路的电压摆幅
    • 因为输出会因为阈值下降而损耗，不能将一个传输门的输出连接到另一个传输门的输入实现串联
    • 阈值电压损失造成静态功耗
  2. 差分传输门逻辑（DPL）

S1.2 动态CMOS设计

S1.2.1 动态逻辑

  1. 动态CMOS结构
    • a是静态CMOS，b是伪NMOS，c是动态CMOS
  2. 动态逻辑基本原理
    1. 预充电Precharge
    • CLK=0CLK=0CLK=0 时，输出节点Out被PMOS预充电至 VDDV_{DD}VDD​，在此期间NMOS被关断下拉网络不工作（求值FET MeM_eMe​ 评估该阶段是否可以放电，消除了预充电期间的任何静态功耗）
    1. 求值Evaluation
    • CLK=1CLK=1CLK=1 时，PMOS关断，PDN导通。根据输入值和PDN的情况有条件的放电
    • Out=CLK‾+(A⋅B+C‾)⋅CLKOut = \overline {CLK} + (\overline {A·B+C})·CLKOut=CLK+(A⋅B+C​)⋅CLK
    • CLK=1CLK=1CLK=1，输入从1跳变0时，输出不变，因为无法拉高
    1. 输入\输出特性
    • 只有预充电阶段可以改变输出
    • 求值阶段输入最多可以一次改变
    • 求值阶段输出可以在高阻抗状态（如果PDN关断）
  3. 动态逻辑的特性
    1. 逻辑功能仅由NMOS下拉网络实现
    2. 晶体管数为N+2
    3. 全电压摆幅（VOH=VDDV_{OH} = V_{DD}VOH​=VDD​, VOL=GNDV_{OL} = GNDVOL​=GND)
    4. 无比逻辑：器件尺寸不影响逻辑功能
    5. 高速开关速度
    • 减少了负载电容（减少了每个门的晶体管数目）
    • 动态门没有短路电流
    1. 只有动态功耗
    • 没有 VDDV_{DD}VDD​ 到 GNDGNDGND 的静态路径
    • 没有毛刺（glitching）
    • 高传输可能性
    • CLKCLKCLK 存在功耗
    1. 需要预充电/求值时钟
  4. 动态设计的问题
    1. 问题1: 电荷泄漏（Charge Leakage）
    • 来源：
      1 NMOS下拉器件A的反偏二极管
      2 亚阈值电流
    • 因此动态电路需要一个最低的时钟频率
    1. 解决方案1：静态补偿电路
    • 类似于PTL的正向反馈方法
    • 为了避免尺寸问题与相关静态功耗，bleeder电阻要大（器件小）
    1. 问题2: 电荷分享（Charge Sharing）
    • 预充电阶段，所有输入为0，且 CLC_LCL​ 被充电至 VDDV_{DD}VDD​
    • 求值阶段，B输入不变，A输入从0跳变至1
    • 原本分配在 CLC_LCL​ 上的电荷就会被分配给 CAC_ACA​ 和 CBC_BCB​
    • 两种可能发生的情况：

    • 用 CLKCLKCLK 驱动的MOS给节点充电
    1. 问题3: 电容耦合（Backgate Coupling）
    • 动态逻辑驱动的静态逻辑，输入In的跳变回影响Out1

S1.2.2 串联动态门

  1. 直接串联动态门形成多级逻辑结构的方法不可行（会使动态逻辑的设计复杂化）
    • Out2会在Out1到达栅极前提前放电，并保持在一个中间电平，导致逻辑错误
  2. 多米诺逻辑Domino Logic
    • 忽略电荷分步与泄漏效应的影响，反相器可以驱动一个泄漏器件以抵抗漏电和电荷的重新分布，并且减少电容
    • 输出加反相器，保证预充电阶段结束时所有的输入都置为0，求值阶段要么保持0，要么翻转到1
    • 扇出由一个具有低阻抗输出的静态反相器驱动提高了抗噪声能力
    • 多米诺CMOS特点：
      • 只能实现非反相逻辑
      • 适用于复杂的大扇出电路（如ALU或者其他控制模块）
      • 可以达到非常高的速度：只存在上升沿延时，静态反相器可以调整用来适应扇出
      • 可以在静态反相器中加bleeder以抵抗电荷效应

    • 无足多米诺Footless Domino
      • 预充电期间多米诺门的输入在低电平，可以取消求值FET，因为可以减少时钟负载并提高下拉驱动能力
      • 取消求值FET会延长预充电时间
      • 考虑上图，在求值期间，In1是1时，Out都是0，In经过反相器都为1，但电平反转的delay一直累积
    • 解决多米诺逻辑非反相的问题
      • 多米诺逻辑的一个主要局限就是只能实现非反相逻辑
      • 可以利用De Morgan定律重新组织逻辑
      • 常用的方法是差分逻辑————双轨多米诺（Dual-rail Domino）类似于DCVSL
  3. np-CMOS
    • 有两种类型（n型树 & p型树）的动态逻辑，避免在关键路径中由多米诺逻辑引入的额外静态反相器
  4. Compound Domino
  5. Multiple-Output Domino Logic(MODL）
  6. Clock-Delayed(CD) Domino

S1.3 总结

  1. 静态 VS 动态
    • 静态：
      噪声情况下鲁棒性
      设计过程无阻碍，易于自动化
      对于较大扇入的复杂门需要在面积和性能上权衡
      伪NMOS简单快速的代价是降低噪声容限和增加静态功耗
      PTL可以用于特定电路，比如多路复用器，XOR主导的逻辑如加法器
    • 动态：
      高速复杂门
      代价是寄生效应，比如电容共享电荷泄漏
  2. 总结
    • 静态互补型CMOS将PDN和PUN相结合，在任意时刻只有一个使能。扇入对CMOS门的性能有很大的影响。处理扇入的技术包括晶体管尺寸，输入重排序和分区。速度也是扇出的线性函数。对于大规模的扇出，需要额外的缓冲
    • 有比逻辑由一个主动下拉(上)网络连接到一个负载设备，大大降低了门的复杂性。在静态功耗和动态功耗之间进行权衡不对称反应。谨慎的晶体管尺寸需要保持足够的噪声空间。常用的技术是伪NMOS和DCVSL
    • PTL实现了一个逻辑门作为一个简单的交换网络。简单的实现。长由于延迟的二次增长，避免了级联开关。nMOS-only PTL生产更简单的结构，但可能遭受静态功耗和减少噪音的利润率。使用电平储存晶体管
    • 动态逻辑是基于电容节点上电荷的存储和条件该节点的放电作为输入的函数。两阶段方案，先预充电后需要求值。多米诺逻辑为性能权衡了噪声裕度。对寄生效应敏感比如漏电，电荷再分配和时钟馈通。级联动态门应该小心处理

S2.2.1 双稳态原理Bi-Stability

  1. 静态储存器用正反馈建立双稳态电路，即电路有两个稳定状态，分别代表0和1
  2. 当翻转区反相器的增益大于1，只有A和B是稳定工作点，C是一个亚稳态（metastable）工作点
  3. 亚稳态：假设交叉耦合的逆变器在C点有偏置，偏离该偏置点的一个小偏差（可能是由噪声引起的）在环路周围被放大和再生(增益> 1)。该偏差被第一个逆变器的增益放大，这个放大的发散被第二个逆变器放大，然后偏置点远离C直到到达其中一个工作点A或B
  4. 双稳态：相反，A和B是稳定的工作点。在这些点上，环路增益比单位小得多。即使与作业点有较大的偏差也会减小并消失。因此，两个逆变器的交叉耦合产生双稳电路，即具有两个稳定状态的电路，每个稳定状态对应一个逻辑状态，例如触发器。这个电路用作存储器，存储1或0(对应位置A和B)。
  5. 触发：为了改变存放的值，我们必须能把电路的状态从A变为B，反之亦然。由于稳定性的先决条件是环路增益小于1，所以可以通过使G增加到一个大于1的值使A（或B）暂时不稳定来改变状态。一般的做法是在 Vi1V_{i1}Vi1​ 或 Vi2V_{i2}Vi2​ 处加一个触发脉冲。例如，假设系统正处于A（Vi1=0，Vi2=1），迫使Vi1变为1会引起两个反相器同时导通一小段时间，并使得环路增益大于1.正反馈再生了触发脉冲的效应，于是电路会以移向另一个状态。触发脉冲的宽度只需稍微大于沿该电路环路的传播延时，也就是反相器平均传播延时的2倍
  6. 改变状态两种方法：
    • 切断反馈回路（普遍使用） 一旦反馈回路打开，一个新的值很容易写入Out（Q），这样的锁存器称为多路开关型锁存器 Q=CLK‾Q+CLKInQ = \overline {CLK}Q+CLKInQ=CLKQ+CLKIn
    • 触发强度超过反馈环 通过在输入触发器，一个新的值被强制进入单元覆盖存储值

S2.2.3 主从边沿触发寄存器Master-Slave Edge-Triggered Register

  1. 一个负锁存器（主）和一个正锁存器（从）级联成的寄存器
  2. 在时钟低位主级是transparent，D传递到主级输出Qm，从级通过反馈保留原有值
  3. 时钟的高位，从级在主级输出采样，同时主级在hold模式
  4. 因为时钟高位时Qm是恒定的，每个周期Q只变一次
  5. 传输门实现
  6. 降低MS寄存器的时钟负载
    • 传输门寄存器的缺点是时钟信号的电容负载很大，这直接影响时钟网络的功耗
    • 可以采用交叉耦合反相器省去反馈传输门
    • 二级可能会影响一级的状态，发生反向传导
  7. 非理想时钟信号
    • 时钟偏差clock skew：布置两个时钟信号的导线会有差别，或者负载电容因储存在所连接的锁存器中的数据不同而变化对时钟信号造成的影响
    • 可能会造成两个信号重叠 Clock Overlap
    • 基于多路寄存器的主要优点是在采样期间反馈回路是打开的。然而，如果 CLKCLKCLK 和C‾LK\overline CLKCLK 之间存在时钟重叠，节点可以由D和B驱动。
    • 这些问题可以通过使用两个不重叠的时钟( ϕ1\phi_1ϕ1​ 和 ϕ2\phi_2ϕ2​ )和保持时钟之间的非重叠时间( tϕ12t_{\phi12}tϕ12​ )足够大，以便即使在时钟路由延迟的存在下也不会发生重叠来避免。在非重叠时间内，寄存器将处于高阻抗状态，如果这种情况持续太长时间，漏电可能会破坏状态

S2.2.4 静态SR触发器————用强信号直接写数据

  1. 压倒反馈回路————交叉耦合晶体管对（NOR）
  2. SR (set-reset)触发器类似于交叉耦合逆变器可以用NOR实现取代逆变器
  3. NOR门的第二输入端连接到触发器输入(S和R)使得强制输出 QQQ 和 Q‾\overline QQ​ 到给定状态
  4. 输出是互补的(除了SR=11状态)
  5. 如果对S输入施加一个正脉冲，则Q输出为强制进入1状态
  6. R上的1信号复位触发器并且Q的输出为0
  7. 交叉耦合晶体管对（NAND）

S2.3 动态锁存器和寄存器

S2.3.2 C2MOSC^2MOSC2MOS————一种对时钟偏差不敏感的方法

  • Clock CMOS
  1. CLK=0CLK = 0CLK=0 (CLK‾=1\overline {CLK} = 1CLK=1):第一个三态驱动器打开，主级作为逆变器对内部节点x上的反向D进行采样。主级处于求值模式，从级处于高阻抗模式(或保持模式)。M7和M8都关闭，将输出与输入D解耦。输出Q保留其先前的值存储在输出电容 CL2C_{L2}CL2​ 上
  2. 当CLK=1CLK = 1CLK=1时，主级处于保持模式(M3-M4关闭)，而第二部分求值(M7-M8开启)，存储在CLC_LCL​上的值通过从级传播到输出节点，从级充当逆变
  3. 对overlap不敏感（只要时钟边的上升和下降时间足够小）
    • 对于(0-0)overlap的情况，两个PMOS器件都在这个时间段，由于NMOS M7关闭，输入数据不能传播到输出
    • 对于(1-1)overlap区域，两个NMOS设备(M3和M7)都是开的，但当M8是关的时，输入数据不会传播出去
    • 一般来说，C2MOSC^2MOSC2MOS 锁存器对时钟重叠不敏感，因为重叠激活了锁存器的上拉或下拉网络，但从未同时激活它们

S2.3.3 真单相钟控寄存器（TSPCR）

  • True Single-Phase Clocked Register
  1. 对 C2MOSC^2MOSC2MOS 锁存器的重新设计
  2. 当CLK=1CLK=1CLK=1时，锁存器处于透明模式，对应于两个级联的反相器，因此不反转，并将输入传播到输出。
  3. 当CLK=0CLK=0CLK=0时，两个逆变器都被禁用，锁存器处于保持模式;只有PUN处于激活状态，PDN处于非激活状态。由于采用双阶段方法，没有信号可以从输入传播到输出
  4. 锁存器嵌入逻辑
    • 逻辑可以嵌入在TSPC锁存器或插入锁存器之间
    • 动态和静态电路可以自由混合
    • 可以设计简化的TSPC寄存器
    • TSPC锁存器不同的组合可以用来实现各种时序元件
  5. TSPC正沿触发寄存器
